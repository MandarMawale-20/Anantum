# Anantum

A fully offline, voice-first AI assistant built in Python.  
Runs locally on consumer hardware — no cloud APIs, no subscriptions.

---

## What it does

Anantum listens to your voice, transcribes it with Whisper, routes your request through a fast pattern classifier, and responds using a local Gemma 3 GGUF model via llama.cpp. Responses are spoken aloud with Kokoro TTS.

It handles two modes:

- **Normal mode** — instant local tools (time, timers, notes, calculator, weather, system info) plus LLM conversation.
- **Celestial mode** — multi-step task planning using a JSON plan generated by the LLM, with web search via DuckDuckGo.

Persistent memory works across sessions using a three-tier system: a hot in-memory cache, a FAISS HNSW semantic index, and a SQLite cold archive.

---

## Hardware requirements

Tested on a machine with an NVIDIA GTX 1650 (4 GB VRAM) and an Intel i5.  
The Q5\_K\_M quantised model (~850 MB) fits entirely in VRAM with that card.

Minimum specs for a usable experience:
- CPU: 4-core modern x86-64
- RAM: 8 GB
- GPU: optional; CPU-only mode works but inference is slower

```bash
CMAKE_ARGS="-DGGML_CUDA=on" pip install llama-cpp-python --force-reinstall
```

If you only want CPU inference:

```bash
pip install llama-cpp-python
```

## Model setup

Download the quantised Gemma 3 1B model and put it in `anantum/models/`:

```
anantum/models/gemma3-voice-Q5_K_M.gguf
```

The model file is not included in this repository.  
You can get it from HuggingFace: search for `gemma3-voice-Q5_K_M.gguf`.

## Running

Voice mode (default):

```bash
cd anantum
python main.py
```

Text mode (no microphone needed):

```bash
python main.py --mode text
```

Common flags:

```
--mode       voice | text           (default: voice)
--model      path/to/model.gguf     (overrides config)
--gpu        N                      (layers to offload; 0 = CPU only)
--voice      af_bella | af_sky | am_adam | af_nicole
--tts-device auto | cuda | cpu
```

Example — full GPU with a different voice:

```bash
python main.py --gpu 35 --voice af_sky
```

## Configuration

All defaults live in `main.py` under the `CONFIG` dict at the top of the file:

```python
CONFIG = {
    "llm_model":          "models/gemma3-voice-Q5_K_M.gguf",
    "whisper_model":      "ctranslate2-4you/distil-whisper-small.en-ct2-float32",
    "kokoro_voice":       "af_bella",
    "llm_n_gpu_layers":   35,    # 0 for CPU-only
    "llm_n_ctx":          2048,
    "llm_n_threads":      4,
    "silence_threshold":  400,   # tune this if your mic is noisy
    "silence_duration":   1.8,
    ...
}
```

## Project structure

```
anantum/
  main.py              entry point, TTS and STT classes
  agent_brain.py       request routing and mode management
  intent_classifier.py  fast regex-based pre-classifier
  llm_manager.py       llama.cpp wrapper and prompt builder
  memory_system.py     three-tier memory (hot / warm / cold)
  tools.py             all tool functions (time, notes, weather, etc.)
  anantum_data/        persisted memory index and SQLite DB (auto-created)
  models/              put your GGUF file here

requirements.txt
```

## Memory system

Three tiers, each with a different latency/capacity tradeoff:

| Tier   | Storage              | Latency   | Capacity            |
|--------|----------------------|-----------|---------------------|
| Hot    | In-memory deque      | <1ms      | Last 20 turns       |
| Warm   | FAISS HNSW + JSON    | ~5ms      | Up to 5000 facts    |
| Cold   | SQLite FTS           | ~20ms     | Unlimited summaries |

Facts are extracted automatically from user messages using regex patterns (name, location, role, preferences, projects). Very short or blacklisted values are discarded.

## Known limitations

- Whisper hallucinations on short/noisy audio. The VAD filter and hallucination list handle most cases but aren't perfect.
- Context window is 2048 tokens by default. Long conversations get truncated to the last 4 exchanges.
- Celestial mode relies on the LLM producing valid JSON for the plan. If it doesn't, execution falls back to a final\_response step.
- Weather and web search require an active internet connection. Other features work fully offline.

## Contributing

Pull requests are welcome. A few things to keep in mind:

- Keep the offline-first constraint. Don't add anything that makes the core tools require internet.
- New tools go in `tools.py` with the `@ToolRegistry.register()` decorator.
- New intent patterns go in the `PATTERNS` list in `intent_classifier.py`, ordered from most specific to most general.
- Keep pattern tests alongside any regex changes if you add them.
